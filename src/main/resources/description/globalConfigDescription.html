Documentation:

Use case one (simple)

The name is the variable name, and the content is the variable value
Example of use: use ${variable name} to refer to variables directly in the template

Use Case Two (Complex)

The name is the variable name, and the content is the template content
Example of use: Use ${variable name} directly to import the template anywhere in the template

Note: Global variables have priority and will affect any variable with the same name in the template.
So be sure not to define a variable with the same name as a global variable in the template.
